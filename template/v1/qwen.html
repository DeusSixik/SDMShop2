💡 Концепция дизайна

  • Компактность: Плашка имеет фиксированный размер и не растёт в зависимости от количества компонентов.
  • Главная информация сверху: Иконка предмета + название (генерируется из reward_item или задается вручную).
  • Бейджи справа: Короткие индикаторы условий (condition_limiter, condition_cooldown) и акций (promo_*).
  • Секция "Получишь": Все reward_* компоненты (иконки + количество). Если наград много - показываем только первые N с "...
  • Секция "Цена": Все cost_* компоненты. Если указан group_id - показываем списком как альтернативы ("ИЛИ").
  • Скидки: При сработавшем discount зачеркиваем старую цену и показываем новую зеленым цветом.
  • Tooltip (подсказка): При наведении мыши на плашку показываем детальный список всех NBT-тегов, полные тексты команд и т.д.
  • Автогенерация имени: Если в JSON не указан display_name, парсим первую награду и берем имя предмета.

🔧 Рекомендации по реализации в Java

  • Создай класс OfferDisplayData, который принимает список компонентов и собирает их в плоскую структуру для рендера.
  • Используй паттерн Visitor для компонентов, чтобы каждый компонент сам решал, как ему отобразиться.
  • Раздели UI на 2 части: Grid View (эта плашка для списка) и Detail View (полная страница товара по клику).
  • Добавь в JSON товара опциональное поле "display" с кастомными параметрами UI (иконка, цвет фона, название), чтобы можно было переопределить автогенерацию.
